# Backend

FastAPI + SQLAlchemy 2 (async) + Celery + Postgres + Redis + Qdrant + MinIO. Houses the API, background workers, AI orchestrator, and RAG indexing pipeline.

For the high-level system view, see [../docs/architecture.md](../docs/architecture.md). For local dev workflow (tests, migrations, codegen), see [../docs/development.md](../docs/development.md).

## Running locally (without Docker)

The full stack expects Postgres, Redis, Qdrant, MinIO, and `unstructured-api` running. The easiest path is `docker compose up postgres redis qdrant minio unstructured-api` for infra and running the API + worker on the host:

```bash
cd backend
uv sync
cp .env.example .env
uv run alembic upgrade head
uv run uvicorn app.main:app --reload
# in another shell:
uv run celery -A app.workers.celery_app worker -Q uploads,ai,embeddings,celery -l info
```

## Tests

```bash
cd backend
uv run pytest -v
```

Tests use an async Postgres test database with per-test transaction rollback. The mock AI provider (`AI_PROVIDER=mock`) returns canned analyze/generate output so the orchestrator can be exercised without external API keys.

---

## Migrations

```bash
uv run alembic revision -m "describe_change"   # autogenerate edit, then:
uv run alembic upgrade head
```

When changing models that touch RAG or AI tables, also bump `embedding_version` on `document_chunks` and run `scripts/reembed.py` if vectors need to be regenerated.
